Formulários e Campos

Formulários são campos onde o usuário irá inserir dados que serão enviados ao back end.

É estritamente proíbido aninhar um formulário dentro de outro formulário. Isto pode fazer com que os formulários se comportem de maneira imprevisível baseada no navegador que está sendo utilizado.

Os tipos de input:

Os formulários recebem vários tipos de inputs diferentes. Eles irão se comportar diferentemente conforme os tipos inseridos neles. Se você precisa, por exemplo, do nome de uma pessoa, utilize o type text. O name é mais utilizado no back end. O id e o name também servem como seletores.

O type indica para o navegador qual o tipo do input.

O placeholder seria o texto de exemplo.

Mais importante, é o value. Value significa o valor do input. Então TODOS os inputs possuem algum tipo de valor. Esse valor é o que vai chegar junto com name no servidor. No caso abaixo, o value não aparece no input de nome porque o valor dele já é o que o usuário irá digitar. Já no input da checkbox, ele aparece como sim, para verificar se é ou não um dev. Neste caso, como não da para digitar nada, o valor que irá aparecer no backend é o valor em value, que no caso é "sim".

O label é o rótulo do formulário. É um elemento em linha, porém ele pode ser transformado em bloco pra quebrar a linha. Além disso, ao clicar nele, ele já seleciona o input que estiver dentro dele ou que estiver baseado no id do input. Por isso o atributo for, dentro do label.

Os tipos de botão

Existem três tipos de botão: botão normal, e os dois dentro do form o reset e o submit. Reset serve para resetar o input e voltar o value dele ao valor padrão. Já o submit, envia o formulário.
